      Attack Trees

      Cyber-only systems, cyber-physical systems, and pure physical systems can all be modeled using attack trees, one of the oldest and most widely used methods for modeling threats. The attack tree method was initially applied standalone, but has since been incorporated into other methods and frameworks, such as STRIDE, PASTA, and CVSS.

      Attack trees are diagrams depicting attacks on a system in tree form; the root represents the attack's goal, and the leaves indicate the methods it uses to accomplish that goal. Each goal should have a different tree, resulting in a set of attack trees that represent the threat analysis.

    Database Design and Implementation

    Database information systems is the most common types of data or databases asit is called in operations, which is an organized collection of structured information, or data, typically stored electronically in a computer system. To explainn further there are different phase of database design.

    Conceptual design

    A conceptual database plan should be created as soon as all data requirements are stored and analyzed. The conceptual database model is highly layered. During this phase, the conceptual design is developed. A basic data modeling operation can be implemented in the conceptual design phase to identify and define the high-level user operations that are evident during analysis of the functions.

    Logical Design

    Similarly, the data modeling mapping phase is known as the logical phase of database design. Through this phase, we attain the result of the relation schema. Class diagrams serve as the basis for these schemas. The process of creating relation schemas is mainly mechanical. Rules exist for converting an ER model or class diagram into a relation schema.

    Normalization

    Database normalization is the process of changing the relation schema to reduce any superfluous information. During each phase of normalization, a new table is added to the database.

    Physical Design

    Physical design is the last phase of database design. The database design is implemented during this phase. DBMS (Database Management System) is required here.

    STRIDE Threat Model Examine each component’s vulnerability to threats and has the following characteristics and breaches of basic security standards.

    SPOOFING

    Spoofing refers to the act of posing as someone else, where an attacker can forge its identity to masquerade as another by creating an illegitimate identity like falsifying or modifying its MAC address to another device. E-mail spoofing is used to lure patients into disclosing personal information or harvesting patients’ credentials. Through tactics like de-authentication attack, where the attacker forges its sender address, a technique commonly used to hide the origin of their e-mails and manipulate accounts to maintain access to victim systems.

    MITIGATIONS

    • Multi-factor authentication: Using a strong password policy.

    Patients must be authenticated by the software.

    • Utilise anti-spoofing and e-mail verification components to channel messages based on legitimacy checks of the sender space and integrity of messages.

    • Active monitoring of social media activities related to your organisation.

    • Segment operational assets to ensure strict isolation of critical control information within the control environment.

    • User Training: staffs should be trained to identify social engineering techniques attempts.

    • Making e-mail secure, the secure socket layer (SSL) is a safe way of sending encrypted information, which is more secured than a conversational paper-based referral that an attacker can intercept.
