    • Abstract

The appointment scheduling system tracks appointments, reschedule appointments based on constraints, resolves conflicts, and manages all interactions. It allows patients to check and communicate with their doctors at their convenience. An online appointment management system allows patients to register and book their appointments with their doctors, and the system best determines which specialist is available in each case. This system helps lessen the burden of waiting for information at the help desk or the clinical staff.     STRIDE Threat Model Examine each component’s vulnerability to threats and has the following characteristics and breaches of basic security standards.

    SPOOFING

    Spoofing refers to the act of posing as someone else, where an attacker can forge its identity to masquerade as another by creating an illegitimate identity like falsifying or modifying its MAC address to another device. E-mail spoofing is used to lure patients into disclosing personal information or harvesting patients’ credentials. Through tactics like de-authentication attack, where the attacker forges its sender address, a technique commonly used to hide the origin of their e-mails and manipulate accounts to maintain access to victim systems.

    MITIGATIONS

    • Multi-factor authentication: Using a strong password policy.

    Patients must be authenticated by the software.

    • Utilise anti-spoofing and e-mail verification components to channel messages based on legitimacy checks of the sender space and integrity of messages.

    • Active monitoring of social media activities related to your organisation.

    • Segment operational assets to ensure strict isolation of critical control information within the control environment.

    • User Training: staffs should be trained to identify social engineering techniques attempts.

    • Making e-mail secure, the secure socket layer (SSL) is a safe way of sending encrypted information, which is more secured than a conversational paper-based referral that an attacker can intercept.

    TAMPERING

    Is a security threat that refers to the unauthorised modification of data to gain control over a system or database. Tampering could either be:

    (1) Physical Tampering and

    (2) Network Tempering,

    Both Physical and network tempering deal with intentionally modifying data. The threat actor intends to alter data in transit, data at rest or data in use; an unauthorised intruder could gather information, intercept a data packet, and change its destination data in transit, with data at rest its modifying data.

    MITIGATIONS

    Monitor and evaluate electronic transactions to support the integrity of the network and application

    • Secure protocols for communications must be put in place to ensure the integrity and confidentiality of all communication at Queens Medical Centre.

    Is a data breach or information leakage; this allows malicious attackers to gather valuable information and reveal sensitive information of a patient’s medical history after obtaining valuable knowledge through active reconnaissance (attack.mitre.org, n.d.). Technological innovation presents a unique security challenge where Mobile devices are difficult to control in any organisation network because most are utilised for personal communications and can be linked to vulnerabilities like unpatched software installed on the mobile device or theft of the device that might land on the hands of adversaries creating a point for credential harvest.

    • Ensure that all sensitive data is encrypted

    DENIAL OF SERVICE

    Is a type of cyber-attack that aims to bring a system or network to a halt, rendering network resources unusable or inaccessible to their intended users. Here attackers intend to Disrupt the proper functioning or the availability of the appointment system by targeting the infrastructure with a flood of Internet traffic and distract the incident response team. Attackers identify the vulnerability in the network through multiple compromised systems and which systems can be exploited and controlled remotely to exploit the clinical security system.

    MITIGATION

    • Ensure step-by-step instructions on reacting to a DDoS attack and ensure Endpoint security that provides network endpoints.

    • Web security tools should be installed to remove web-based threats, block abnormal traffic, and search for known attack signatures.

    • Network segmentation should be implemented to protect the network from DDOS attacks (separates systems into subnets)

    • Continuous Monitoring of Network Traffic

    ELEVATION OF PRIVILEGES

    Elevation of privilege attacks allegedly uses security flaws to gain access to a network and applications. The attacker tries to gain access to a lower account like a user through phishing attack and uses the compromised account to gain a higher-level privilege. This could be described as the most threatening of the STRIDE framework as users, being the weakest link in the security chain. This can be possible over a regular user’s account on the network and then attempts to gain administrative permissions. To ultimately steal users’ information from the data base of the ASMIS.

    MITIGATION

    • Applications with known vulnerabilities should not have set user identity on execution or the "set user identity. bits" set to reduce the potential damage if they are compromised.

    Users should be removed from the local administrator group on systems and the most pressing data protection issues should be addressed.

    • Monitor the file system for files that have malicious signatures.

    • Files should be encrypted such that passwords are always required. (attack.mitre.org, n.d.)

    • “Develop a robust cyber threat intelligence capability to determine what types and levels of threat may use software exploits and 0-days against a particular organization”. (attack.mitre.org, n.d.)
